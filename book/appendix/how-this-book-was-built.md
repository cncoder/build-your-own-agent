# 附录 A：本书是怎么用 AI Agent 写出来的

---

## 1. 团队架构

这本书不是一个人写的，也不是一群人写的——它是一个 **Agent Team** 协作的产物。

```
┌─────────────────────────────────────────────────────────────────┐
│                        总编辑 Agent (INTJ)                       │
│              协调全局、分配任务、收集反馈、触发重写              │
└──────────────────────────────┬──────────────────────────────────┘
                               │ 调度
         ┌─────────────────────┼─────────────────────┐
         │                     │                     │
┌────────▼────────┐  ┌────────▼────────┐  ┌────────▼────────┐
│  研究 Agent ×3   │  │   编辑 Agent     │  │  多媒体 Agent    │
│                  │  │                  │  │                  │
│ • 架构分析       │  │ • Markdown 整合  │  │ • 播客脚本生成   │
│ • 生态调研       │  │ • 风格统一       │  │ • TTS 合成       │
│ • 社区热点       │  │ • 编辑铁律执行   │  │ • PPT 制作       │
│ • 100+ 仓库阅读  │  │ • 证据层级把关   │  │ • HTML Demo      │
└─────────────────┘  └─────────────────┘  └─────────────────┘
         │                                          │
         │                                          │
┌────────▼────────┐                       ┌────────▼────────┐
│  学生 Agent      │                       │  评分 Agent      │
│  (INFP 视角)     │                       │  (ISTJ 视角)     │
│                  │                       │                  │
│ • 零基础读者模拟 │                       │ • 6 维度打分     │
│ • "看不懂"反馈  │                       │ • 名师范式对齐   │
│ • 啊哈时刻检测   │                       │ • ≥9 门控        │
└─────────────────┘                       └─────────────────┘
```

每个 Agent 有明确的人格设定（MBTI 类型）、独立的 prompt、独立的评判标准。学生 Agent 会说"这段我看不懂"，评分 Agent 会说"叙事节奏低于 Karpathy 标准，不通过"——两者的反馈都会触发总编辑重新分配修改任务。

**关键设计决策**：评分 Agent 和写作 Agent 严格分离。写作 Agent 不能给自己打分——这是防止 LLM 自我评价过度乐观的结构性约束。

## 3. 反思：Agent 擅长什么，不擅长什么

### Agent 比人强的地方

**批量一致性**。26 章都有 README + code + ppt.md + podcast.md + demo，格式完全统一。人类写 26 章，到第 15 章格式就开始走样了。Agent 不会疲劳。

**速度**。72 小时产出 60 万字 + 配套媒体。同样质量的内容，纯人工写作估计需要 3-6 个月。Agent 的价值不是替代思考，而是替代重复劳动——把"想清楚了之后的执行"压缩到极致。

**交叉验证**。学生 Agent 和评分 Agent 同时审同一章，视角完全不同。人类审稿时很难同时保持"我是初学者"和"我是严格评委"两个视角，Agent 可以。

### 必须人兜底的地方

**审美判断**。"这章读起来有没有节奏感"——Agent 可以用 rubric 打分，但最终判断是人的。评分 9.2 和 9.5 之间的差距，往往是"某段话读起来有没有画面感"这种无法量化的东西。

**全书连贯性**。Agent 写单章很强，但"26 章串起来是不是一个连贯的故事"需要人从高处俯瞰。Lena 从 v0.1 到 v0.25 的成长弧线，是人设计的，不是 Agent 自己涌现的。

**不犯低级错误**。Agent 偶尔会在正文里泄露本机路径、写出作者真名、或者在理论章里突然插入过多案例。这些需要编辑铁律 + 人类最终 review 的双重保障。

### 下次会改什么

1. **前期花更多时间设计全书叙事弧线**，而不是让 Agent Team 直接开写。先有 spine（脊柱），再长 flesh（肉）
2. **评分 Agent 加入"章间一致性"维度**——不只看单章质量，还看相邻章节的衔接是否自然
3. **TTS 质检前置**——播客脚本生成时就用 ASR 模拟验证，而不是等 mp3 出来再发现问题
