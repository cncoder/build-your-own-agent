03
核心概念
agent = LLM + loop + tools
🔁
Loop（while 循环）
让决策不止一步。每次工具调用后，把结果带给 LLM 继续推理。
🧠
LLM（大脑）
决定下一步做什么——用工具还是直接回答。
🔧
Tools（双手）
LLM 做不了的事——查时间、读文件、搜网页。
用户输入 │ messages[] 加入用户消息 │ ▼ while True: │ ├─ provider.chat(messages, tools) │ │ │ tool_use? ──是──▶ execute_tool() │ │ │ │ │ 否 tool_result 加入 messages │ │ │ │ └──────────────────┘ │ └─ stop_reason == "end_turn" ──▶ 返回文字答复

05
工具系统
工具 = schema + handler
为什么需要 schema？
LLM 无法直接调函数。
它能做的：输出一个 JSON 块说"我想调用这个函数"。

Schema = 交给 LLM 的"工具使用说明书"
Handler = 真正执行的 Python 函数
input_schema = JSON Schema
这是 RFC 标准格式，Pydantic 可以自动生成
（Ch 4 会讲）。本章手写是为了理解细节。
# tools.py（核心片段） TOOLS = [ { "schema": { # ← 给 LLM 看 "name": "get_time", "description": "获取当前本地时间", "input_schema": { "type": "object", "properties": { "timezone": { "type": "string", "default": "local" } }, "required": [] } }, "handler": get_time_handler # ← 给 Python 运行 } ]
06
协议差异
Anthropic vs OpenAI：工具调用格式
🔷 Anthropic
// 工具调用在 content[] 里
content = [
  {"type": "tool_use",
   "id": "toolu_xxx",
   "name": "get_time",
   "input": {}}
]

// 结果回填：role = "user"
{"role": "user",
 "content": [{
  "type": "tool_result",
  "tool_use_id": "toolu_xxx"}]}
🔶 OpenAI
// 工具调用在顶层字段
message.tool_calls = [
  ToolCall(
    id="call_abc",
    function=Function(
      name="get_time",
      arguments="{}")
  )
]

// 结果回填：role = "tool"
{"role": "tool",
 "tool_call_id": "call_abc",
 "content": "..."}
对比项AnthropicOpenAI
工具调用位置content[] 数组顶层 tool_calls 字段
参数字段input（dict）arguments（JSON 字符串）
结果回填角色usertool
关联 ID 字段tool_use_idtool_call_id
07
核心实现
AgentLoop 核心——28 行
def agent_loop(user_input, provider, max_turns=10): messages.append({"role": "user", "content": user_input}) tools = get_tool_schemas() for turn in range(max_turns): response = provider.chat(messages, tools) if response.tool_calls: # 1. 存入 assistant 消息（含工具调用决策） messages.append(provider.make_assistant_with_tool_use(response)) # 2. 执行工具，回填结果 for tc in response.tool_calls: result = execute_tool(tc["name"], tc["inputs"]) messages.append(provider.make_tool_result_message(tc["id"], result)) continue # 再次问 LLM # 无工具调用：返回最终答复 messages.append({"role": "assistant", "content": response.content}) return response.content
✓ 这就是 agent 循环的全部秘密。所有 agent 框架的核心都是这 28 行。
08
执行流程
一次"现在几点"的完整时序
步骤 发起方 目标 内容
1 👤 用户💬 messages[] "现在几点了？"
2 🔁 AgentLoop🧠 LLM chat(messages, tools=[get_time])
3 🧠 LLM🔁 AgentLoop stop_reason="tool_use" content=[tool_use{name="get_time"}]
4 🔁 AgentLoop⚡ get_time() execute_tool("get_time", {})
5 ⚡ get_time()🔁 AgentLoop "当前时间是 2026年05月06日 00:24:31"
6 🔁 AgentLoop🧠 LLM chat(messages + tool_result)
7 🧠 LLM👤 用户 stop_reason="end_turn" "现在是 2026年5月6日 00:24 🌙"
⚡ LLM 被调用了 2 次：第一次决策工具，第二次生成答复

实战验证
跑通的样子
$ python lena.py --provider bedrock Lena v0.3 ✦ provider=bedrock 输入 'exit' 或按 Ctrl-C 退出 你：现在几点了？ [工具] get_time({}) → 当前时间是 2026年05月06日 00:24:31（local） Lena：现在是 **2026年5月6日 00:24**， 已经是深夜了哦！🌙 你：今天是星期几？ Lena：2026年5月6日 是 星期三！📅 （没有再次调工具，从上下文推断） 你：exit 再见！
✓ 注意第二个问题
问"星期几"时，Lena 没有再次调用 get_time。

因为 messages[] 里已有时间，LLM 从上下文推断——这是 Memory 的价值：减少不必要的工具调用。

本章小结
五条金句
① agent = LLM + loop + tools。掌握这个三元组，你就掌握了所有 agent 框架的本质。
② MVA 6 模块（Config / Provider / Memory / ToolRegistry / AgentLoop / Skills）是行业共识。三个开源实现都有对应体。
③ tool_use 是 agent 的神经接口。Anthropic 和 OpenAI 格式不同，但逻辑相同：输出意图 → 执行 → 回填。
④ messages[] 是唯一的真相源。用户输入、LLM 决策、工具结果——全在这个列表里。
⑤ 先裸写，后框架。等你写通 50 行，再看 LangChain 的源码，你会发现它只是这 50 行的装饰版本。
