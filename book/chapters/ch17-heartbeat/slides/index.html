Proactive vs Reactive

Reactive

维度状态
触发方式用户发消息
信息主动权在用户
适用场景搜索、Q&A
本质工具
局限用户不问 = 没有输出

Proactive

维度状态
触发方式Heartbeat 定时触发
信息主动权在 Agent
适用场景日报/预警/进度汇报
本质助理
优势Agent 决定何时说什么

Design Note： Anthropic 2026-04「Scaling Managed Agents」报告将 proactive vs reactive 列为当前 agent 系统的核心议题。 主动性是 2026 年 agent 能力边界最重要的话题。

nano-claw Heartbeat · 178 行甜点级

setTimeout + EventEmitter

// nano-claw/heartbeat/index.ts

class HeartbeatTimer {
  start(onTick: () => void): void {
    const tick = () => {
      onTick();
      // 递归 setTimeout，保证不重叠
      setTimeout(tick, this.intervalMs);
    };
    setTimeout(tick, this.intervalMs);
  }
}

class HeartbeatRunner extends EventEmitter {
  private async onTick() {
    if (!isActiveHours(this.config)) return;
    const payload = await this.generatePayload();
    if (payload) this.emit("outbound", payload);
  }
}

为什么 setTimeout 不用 setInterval？

setInterval 固定间隔触发——
如果执行花了 30s，下次可能立即触发。

递归 setTimeout = 上次完成后才开始计时，
保证节拍不重叠。

3 个核心组件

① Timer setTimeout 递归节拍器
② ActiveHours 时区感知过滤器
③ EventEmitter 推送事件总线
