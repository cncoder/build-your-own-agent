核心概念 02
LLM vs Agent：函数 vs 程序
f(x) LLM（函数）
✗
无状态
每次调用独立，不记得上次说了什么
✗
无工具
只能生成文字，不能查 API、读文件
✗
无循环
一问一答，不能多步规划
✓
极速
单次调用毫秒级，cost 低
# 函数模型
result = llm("明天天气?")
# 结束了，没有循环
⟳ Agent（程序）
✓
有循环
while True: 多步决策，直到任务完成
✓
有工具
调用 API、读文件、执行代码、搜索网页
✓
有记忆
每步结果加进上下文，上一步影响下一步
✓
有主动性
不等你问，定时推送晨报（Ch13 Heartbeat）
# 程序模型
while task.pending:
  decision = llm(context)
  if decision.tool:
    result = execute(decision.tool)
    context += result # 循环！

Anthropic
Messages API
# system 是顶层字段
client.messages.create(
  model="claude-opus-4-5-20251101",
  system="你是 Lena", ← 顶层
  messages=[{
    "role": "user",
    "content": "你好" ← 字符串
  }]

# 回复路径
resp.content[0].text
resp.stop_reason # "end_turn"
OpenAI
Chat Completions
# system 在 messages 里
client.chat.completions.create(
  model="gpt-4o",
  messages=[
    {"role":"system", ← 列表里
     "content":"你是 Lena"},
    {"role":"user",
     "content":"你好"}
  ]
)

# 回复路径
resp.choices[0].message.content
resp.choices[0].finish_reason
AWS Bedrock
Converse API
# content 是列表！blood tears: inferenceProfileId
bedrock.converse(
  modelId="us.anthropic.claude-sonnet-4-6",
  # ↑ inferenceProfile，不是 modelId！
  system=[{"text":"你是 Lena"}],
  # ↑ 列表，不是字符串！
  messages=[{
    "role":"user",
    "content":[{"text":"你好"}] ← 列表
  }],
  inferenceConfig={"maxTokens":1024}
)

# 回复路径（多层）
resp["output"]["message"]["content"][0]["text"]

Design Note
为什么选 Python？——生态成熟，动态工具 schema 最顺手
⚠ Design Note · 权衡讨论
Python ✓ 本书主线
✓LangGraph / CrewAI / pydantic-ai 全是 Python-first
✓Pydantic 动态 JSON Schema，工具定义最顺手
✓后端工程师最大公约数，入门无障碍
✓Bedrock / OpenAI / Anthropic SDK 全覆盖
—GIL 并发，CPU 密集场景不如 Go/Rust
TypeScript ○ CLI agent
✓Claude Code / OpenClaw 用 TS（本书参考实现）
✓前端深度集成（Next.js + agent）
✓Zod schema 类型安全工具定义
—无 LangGraph 等价实现
—动态 tool schema 摩擦较大
Rust △ 附录 A
✓极低延迟代理层（rtk，减少 60-90% token）
✓嵌入式 agent runtime
—无 LangGraph/CrewAI 等价物
—serde_json::Value 动态 schema 摩擦大
—MCP 生态仍处于早期（R3 §四）
