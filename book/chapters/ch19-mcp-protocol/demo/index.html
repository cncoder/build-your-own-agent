Client ↔ Protocol ↔ Server 握手时序

MCP（Model Context Protocol）是标准化的 LLM 工具扩展协议。一次完整会话包含：能力协商 → 工具列表 → 工具调用 → 结果返回。

读者学到什么
MCP 基于 JSON-RPC 2.0，每条消息有 jsonrpc / method / params / id 四个字段。握手分三步：① initialize 协商能力版本；② tools/list 拉取工具清单；③ tools/call 调用具体工具。错误时返回标准 error 对象（code + message），与成功 result 互斥。
