阻挂 docker socket

/var/run/docker.sock → 容器逃逸的经典路径
攻击路径：如果容器挂载了 docker socket
STEP 1
容器内运行攻击者代码
Agent 执行了用户提供的"无害"脚本
↓
STEP 2
docker -H unix:///var/run/docker.sock run ...
通过 docker socket 创建新容器
↓
STEP 3
--privileged -v /:/host
新容器挂载宿主机完整文件系统
↓
RESULT
chroot /host bash → 宿主机 root shell
容器安全边界被完全突破

Design Note
容器访问 Docker daemon = 逃逸容器。
阻断方式：启动前校验，发现挂载 socket 立即拒绝启动。
这是 docker.ts 最重要的 3 行。
