两个真实沙箱案例

沙箱 ≠ 云端——形式可以多样
案例 16.1
cat-vision
本地 MLX 模型沙箱化
cat-vision/cat_monitor.py
RTSP 摄像头 (内网)
→
ffmpeg 截帧
↓
mlx_vlm 本地推理 3-8s/张
→
JSON 行为分类
↓ 异常才
Bedrock Claude（精判）
→
Discord + HA
数据不出内网：所有图像本地处理，不上传
95% 案例本地推理，5% 异常才调 Bedrock
模型首次加载缓存，后续推理无加载延迟
教学点
沙箱 = 控制信息流，不一定是 Docker 容器。
数据不出内网 = 网络级隔离沙箱。
案例 16.2
exposure-monitor
双引擎 Lambda 架构
exposure-monitor/CLAUDE.md
Reactor (秒级)
Full Scanner (900s)
CloudTrail→EventBridge→Lambda(15s)
定时全量扫描
↓
检测 0.0.0.0/0 规则
→
TCP connect 验证可达性
↓ 可达才报警
SNS → Discord
"可达才算暴露"：SG 有规则但 ENI 未 attach 不报
Lambda 本身就是沙箱：函数执行环境隔离，执行完销毁
与 Docker sandbox 同思路：每次独立执行环境
教学点
Serverless 是另一种"每次独立容器"沙箱。
设计思想与 Docker sandbox 一脉相承。
