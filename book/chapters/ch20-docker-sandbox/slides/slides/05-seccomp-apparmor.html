seccomp + AppArmor

系统调用级别的沙箱——一句话说清原理
SC
seccomp
Secure Computing Mode · Linux 内核功能
程序做任何事最终都要调系统调用。seccomp 在这一层设白名单——超出白名单直接被内核杀掉。
Docker 默认启用 seccomp profile，阻断 44 个危险系统调用
包括 ptrace（进程注入）、keyctl（密钥操作）、mount（挂载文件系统）
白名单之外的调用：SIGSYS 终止进程
AA
AppArmor
Mandatory Access Control · MAC 框架
seccomp 是"能不能发起这种调用"，AppArmor 是"这个程序能不能访问这个资源"。两者互补。
为每个进程定义允许访问的文件路径
限制网络操作和 Linux capability
违反 profile 的访问：内核层面直接拒绝
这些配置必须被阻断
--security-opt seccomp=unconfined
允许所有系统调用 → 可能提权/逃逸
--security-opt apparmor=unconfined
解除 MAC 约束 → 任意资源访问
--privileged
超级权限 = 无沙箱
validate-sandbox-security.ts:311-322
