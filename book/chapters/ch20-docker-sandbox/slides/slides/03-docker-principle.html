Docker 容器隔离原理

代码在笼子里跑——出不来就安全了
宿主机 (Host OS)
Docker Engine
seccomp + AppArmor 约束层 禁用系统调用
容器 (Ephemeral)
任意代码执行
CPU ≤ 0.5 core
内存 ≤ 256 MB
网络 = none
文件系统隔离
--rm → 执行后立即销毁
FS
文件系统隔离
容器看不到宿主机文件
根文件系统只读 + tmpfs /tmp
NS
进程命名空间
独立 PID 空间
容器内进程看不到宿主机进程
NET
网络隔离
network=none 完全断网
代码无法发起网络连接
RM
即时销毁 (--rm)
执行完毕容器立即销毁
每次执行都是全新环境
