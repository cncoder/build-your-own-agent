真沙箱 vs VM vs Wasm

三种隔离方案对比——选 Docker 的理由
方案 隔离强度 启动时间 资源开销 适用场景 lena-v1.x
正则黑名单 低（可绕过） 0ms 极低 本地开发 / 信任代码 v1.0 ShellSandbox
Docker 容器 ⭐ 高 0.3s - 2s 中 生产代码执行 / 多租户 v1.6（本章）
虚拟机 (VM) 极高 3s - 30s 高 金融级隔离 / 高安全要求 —
WebAssembly 中-高 < 10ms 低 浏览器 / 边缘计算 —
gVisor (runsc) 很高 500ms 中 兼顾安全与性能 --runtime=runsc 切换
升级路径
lena-v1.6 用 Docker（docker run）→ 升级到 gVisor 只需加 --runtime=runsc，代码不变，隔离强度大幅提升。
