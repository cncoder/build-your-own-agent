class MemoryStore: def __init__(self, db_path): self.db_path = Path(db_path).expanduser() self._init_db() def append_message(self, session_id, role, content): """存 user/assistant/tool_result""" now = datetime.utcnow().isoformat() content_json = json.dumps(content) with self._conn() as conn: conn.execute( "INSERT INTO messages ...", (session_id, role, content_json, now) ) def load_messages(self, session_id): """按顺序读取整段历史""" rows = conn.execute( "SELECT role, content FROM messages WHERE session_id=? ORDER BY id" ).fetchall() return [{"role": r, "content": json.loads(c)} for r, c in rows]
关键设计决策
  • 📦 content 存 JSON：文本或 tool_use blocks 统一格式
  • ⚠️ summary 字段：compaction 摘要存这里——仅调试，不信任
  • 🔢 INTEGER AUTOINCREMENT：保证顺序，不用 timestamp 排序
  • 🔒 WAL 模式隐式启用：多进程安全
⚠️ COMPACTION TRAP
autoCompact 生成的摘要 = LLM 输出 = 可以幻觉。

🔁 RECITATION 技术（Manus）
把关键记忆写在 system prompt 末尾，不只放开头。
LLM 对 context 末尾注意力更强（lost-in-the-middle）。
→ 长会话中偏好不会"漂移消失"。
def _build_system_prompt(self) -> str: base = "你是 Lena，用户的 AI 助手..." # 从文件系统加载长期记忆 long_term = self.memdir.format_for_prompt() if not long_term: return base # Recitation：开头 + 末尾双注入 return ( f"{base}\n\n" f"{long_term}\n\n" # 开头注入 f"\n" f"{long_term}" # 末尾重申 ) # 注入效果 system_prompt = """ 你是 Lena，用户的 AI 助手... ## 已知信息（长期记忆） - programming_language (preference): 用户偏好 Python - user_name (fact): 用户叫 Bob - programming_language (preference): 用户偏好 Python - user_name (fact): 用户叫 Bob """

设计决策
为什么文件系统，不用向量数据库？
向量库的代价（个人 agent 场景）
  • ❌ 嵌入模型依赖（OpenAI embedding / local model）
  • ❌ 索引更新延迟（写后不能立即搜到）
  • ❌ 相似度阈值调参地狱
  • ❌ 召回结果不可解释（为什么召回这条？）
  • ❌ 额外存储服务（Pinecone / Chroma / Weaviate）
文件系统的优势
  • ✅ 零依赖（ls + cat 就能检查）
  • ✅ 可编辑（vim 直接改记忆）
  • ✅ 精确检索（关键词匹配，不是相似度）
  • ✅ git 版本管理记忆变化
  • ✅ 个人 agent 记忆量 < 1000 条，遍历够用
"文件系统是无限外部记忆"
— Manus 团队，Context Engineering Lessons（HN 120 points）
向量库真正适合的场景
  • 📚 大规模语料（10万+ 文档）
  • 🔍 语义相似度（"找类似的文档"）
  • 🤖 RAG 知识库（外部知识检索）
个人 agent 偏好/知识记忆 → 文件系统 ✓
企业知识库 RAG → 向量库 ✓

案例 6.2 + 终端演示
Compaction 不可信事故 & Demo
⚠️ 事故经过
1. 用户明确否决"用 LangChain 重写 agent loop"
2. 会话继续，context 触发 autoCompact
3. LLM 生成摘要，把已否决方案写成"待评估候选方案"
4. Lena 看到摘要，开始重新推进已否决方案
5. 用户发现并叫停
✅ 解法：外部持久化 + 不信摘要
铁律：compaction 后第一件事——逐项对照 MEMORY.md，不信 compaction 的 summary。

重要决策写 MemDir，不依赖会话摘要。
