Design Note · 选型指南
什么时候 fork？什么时候用 multi-agent？
"我要一个专用 agent，还是给通用 agent 切换 context？"
如果你需要长期稳定运行的专用角色（7×24 trading bot），用 fork。
如果是一次性任务（帮我分析这份报告），给通用 agent 换个 system prompt 更轻量。
"专用 agent 之间需要协作，用 Crew 还是 Flow？"
一次性任务（生成报告、分析数据）→ Crew
持续运行（监控 / 日报 / 定时任务）→ Flow
两者都用的话，把 Crew 包在 Flow 的一个 Step 里。
"多个专用 agent，用户怎么知道找哪个？"
SupervisorAgent（agent-as-tools）：用户只跟 Supervisor 交互，Supervisor 负责路由。
优点：用户零认知负担，路由决策是 LLM 推理，可以处理复杂意图。
