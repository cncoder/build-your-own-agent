# Trading SOP ## Step 1：市场扫描（每 5 分钟） - 调用 get_price 获取实时价格 - 调用 get_indicators 获取 RSI(14)、MACD(12/26/9)、BB(20,2) ## Step 2：信号确认（≥2/3 指标共振） 多头：RSI < 60 + MACD 金叉 + BB 中轨以上 空头：RSI > 40 + MACD 死叉 + BB 中轨以下 ## Step 3：风控（代码层硬编码，不可绕过） 单笔敞口 ≤ 总资金 2% 连续亏损 3 次 → 熔断 24h 每日最大亏损 ≤ 5% ## Step 4：执行 1. 记录决策理由到 trade_log.jsonl 2. 调用 place_order 3. 设置止损 -1.5% + 止盈 +3% ## 熔断状态机 CLOSED → (连续亏 3) → OPEN OPEN → (冷却 24h) → HALF_OPEN HALF_OPEN → (1 笔盈利 + 清零计数) → CLOSED
Skills 注入的三大好处
01
可版本管理
SOP 是 markdown 文件，可以 git 追踪变更历史，可以 code review，可以 A/B 测试不同策略版本
02
热加载更新
auto_eval 机制可以在不重启 agent 的情况下更新策略参数，实时生效
03
结构化比 prompt 更好
Step-by-step SOP 比散文式 prompt 更能让 LLM 按顺序执行，减少遗漏关键步骤
血泪教训 [案例 19.1]：auto_eval 热加载会覆盖 config.py——热加载只能更新 eval_params 命名空间，不能碰 base_config。
加 version lock：config._locked = True 后不允许 eval 覆盖。
