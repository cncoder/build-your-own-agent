为主题 **$ARGUMENTS** 生成一份结构化报告，按以下 SOP 处理：

## 数据提取规则

1. 从对话上下文中提取关键数据点（数字、趋势、对比）
2. 数字格式：超过 1000 的整数加千分位（1,234）；金额保留两位小数
3. 如果没有上下文数据，生成一份示例结构展示报告格式

## 报告结构

```
# [主题] 报告
生成时间：YYYY-MM-DD

## 核心结论（3 条，最重要的在最前面）
1. ...
2. ...
3. ...

## 数据概览
[关键数字和趋势，表格或列表形式]

## 分析与建议
[基于数据的 2-3 条可操作建议]
```

## 排版规则

- 每份报告不超过 2 页（约 800 字）
- 数据图表用 ASCII 或 Markdown 表格替代
- 避免堆砌数字，每个数字要有对应解读

## 输出步骤

1. 先在回复中展示报告预览（纯文本）
2. 询问用户是否满意，满意后调用 generate_pdf 工具生成文件
3. 返回文件路径和页数

## 错误处理

- 渲染失败：告知用户"报告生成遇到问题，已保存草稿"，不暴露技术细节
- 内容过长：自动压缩，保留核心结论和最重要的 3 条数据
