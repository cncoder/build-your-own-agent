Skills 文件格式
YAML frontmatter + Markdown 正文
--- name: weather # → /weather 命令 description: 查询城市天气 argument-hint: "<城市名>" # UI 提示 allowed-tools: # 工具白名单 - get_weather tags: - weather - utility --- 查询 $ARGUMENTS 的当前天气。 ## 步骤 1. 确认城市名（别名→标准名） 2. 调用 get_weather 工具 3. 格式化：温度/湿度/风速/趋势 4. 给出出行建议（一句话） ## 错误处理 - API 错误："天气服务暂不可用" - 城市不识别：询问用户确认

Ch 09 · Skills 能力单元 05 / 12 · $ARGUMENTS 参数替换
$ARGUMENTS 参数替换
用户输入 /weather 上海 时：
Skill 正文里所有 $ARGUMENTS
被替换为 上海，然后注入 system prompt。
本质：参数化的 system prompt 片段

Skills 本质上是可复用的 system prompt 模板。
每次调用时，动态展开并追加到当前 system prompt 末尾。
简单、透明、可预期——没有任何黑盒逻辑。

--- name: weather description: 查询城市天气并生成易读简报 argument-hint: "<城市名>" allowed-tools: - get_weather --- 查询 $ARGUMENTS 的当前天气，按以下 SOP 处理： ## 步骤 1. 确认城市名：别名自动转换标准名 2. 调用工具：使用 get_weather 获取数据 3. 格式化输出：温度/体感/描述/湿度/风速/趋势 4. 出行建议：一句话总结 ## 错误处理 - API 错误→ "天气服务暂不可用，请稍后重试" - 城市不识别 → 询问用户并提供相似城市 - 超时 → 提示检查网络 ## 输出格式 🌤 {城市} 天气（{日期时间}） 温度：{temp}°C（体感 {feels}°C） 天气：{description} 湿度：{humidity}% | 风速：{wind} 未来 24h：▲/▼ {forecast} 💡 建议：{advice}

"a bigger deal than MCP"
MCP 解决工具连接的标准化，Skills 解决知识复用——后者生态价值更大，因为知识比工具更难获取、更难标准化。

可共享性

一个专家写的 Skill 可以被所有 agent 复用，跨机器、跨团队、跨项目

互补不替代

Skills 不替代工具。工具提供能力，Skills 提供知识

低门槛扩展

非工程师也能扩展 agent 能力，只需写 Markdown，无需编码、无需部署

生态价值

形成"技能市场"——专家贡献 Skill，所有人受益。类比 npm 包但针对知识

Ch 09 · Skills 能力单元 11 / 12 · Design Note：Skills 版本管理
Skills 怎么防止 SOP 过时？

Design Note

Skills 的 SOP 是"活的知识"——API 可能变，最佳实践可能更新，错误处理方式可能改进。一个写于 2025 年的 Skill，2026 年还可用吗？这不是技术问题，而是知识管理问题。

📋

Frontmatter 版本字段

在 frontmatter 里加 version: 1.2 和 last-updated: 2026-05，让使用者知道 Skill 的新鲜程度

🔄

learn.sh 自动 PR 流

踩坑后立即 bash learn.sh --skill 更新并提 PR，知识的半衰期比你想象的短

📊

skill-usage.jsonl 埋点

每次调用记录成功/失败，每周聚合——高失败率 = Skill 可能过时了

案例 9.1 · ironman-config 实践
# 学到新技能立即同步 $ bash ~/ironman-config/learn.sh \ --skill ~/.claude/skills/my-skill.md \ "技能描述" # learn.sh 自动做： # 1. git checkout -b learn/skill-xxx # 2. cp skill → ironman-config/skills/ # 3. git add + commit + push # 4. gh pr create # 其他机器同步： $ git pull && bash install.sh
血泪教训：断更 27 天

ironman-config 有 2026-03-21 → 2026-04-18 的 27 天空白期。这段时间踩的坑没有同步，后来又遇到同样问题才发现代价。铁律：踩坑当天就用 learn.sh 记录。

案例 9.2 · Voyager 式自进化

Lena 每周日聚合 skill-usage.jsonl，零使用 7 天自动归档。Skills 不是静态的，而是可以演化的。
