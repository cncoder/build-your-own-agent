案例 11.1 · 血泪教训
OpenClaw Gateway 重启风暴
launchd KeepAlive + 端口冲突 = 每秒数次崩溃重启
根因
Gateway 启动 → 端口 8765 冲突 → 崩溃
↓
launchd KeepAlive → 立即重启
↓
端口仍被占用 → 再次崩溃
↺ 死循环
修复
<!-- plist 加 ThrottleInterval --> <key>ThrottleInterval</key> <integer>60</integer> // 60s 兜底 + 应用层退避 // = 双重保护
血泪铁律
launchd KeepAlive: true
不加 ThrottleInterval
= 重启风暴
两层防护 = 健壮系统
launchd ThrottleInterval: 60s
+ 应用层 BackoffPolicy: 1s→5min
→ 服务器安全，进程自愈
案例 11.3 · 安全迁移
Discord Bot Token：config → env var
SIGUSR1 不刷新 plist env var——必须 launchctl unload/load
// 危险：明文 token 在 config { "channels": { "discord": { "token": "MTxxxxxx.Gxxxxx" } } } // Step 1: 提取到 env 文件 DISCORD_BOT_TOKEN=MTxxxxxx.Gxxxxx // chmod 600 ~/.openclaw/.env.discord // Step 2: 写入 plist EnvironmentVariables /usr/libexec/PlistBuddy \ -c "Add :EnvironmentVariables:DISCORD_BOT_TOKEN \ string MTxxxxxx" ~/Library/LaunchAgents/ai.openclaw.gateway.plist // Step 3: 从 config 删除明文 token jq 'del(.channels.discord.token)' \ ~/.openclaw/openclaw.json > /tmp/new.json mv /tmp/new.json ~/.openclaw/openclaw.json // Step 4: 完整重启（不能只 kill/SIGUSR1！） launchctl unload ~/Library/LaunchAgents/ai.openclaw.gateway.plist launchctl load ~/Library/LaunchAgents/ai.openclaw.gateway.plist
⚠️ 血泪教训
SIGUSR1 让 Gateway 重读 config
但 env var 是 launchd 在启动时注入的
SIGUSR1 拿不到新值

必须 unload → load
代码侧读取

Design Note
nano-claw 219 行 vs OpenClaw 230 文件
何时用教学版？何时升级到生产版？
用 nano-claw（教学版）当…
✓ 个人助理，1-2 个 channel
✓ 原型验证，快速迭代
✓ 学习 Gateway 工作原理
✓ 单机部署，不需要监控
升级到 OpenClaw 当…
✓ 接入 5+ channel
✓ 需要完整指标 / Prometheus
✓ 多 agent 协作（researcher / ops）
✓ 生产部署，需要 SLA

OpenClaw 给你 平台
