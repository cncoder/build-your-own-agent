from datetime import datetime

Long-running Task

三大挑战

👁️
进度可见
黑盒跑 24 小时，第 18 小时崩了——
你什么都不知道
反模式： 整个跑完再落盘
正确： 每步落一次 checkpoint
💥
崩溃恢复
网络断了、OOM、机器重启——
进程可能在任何地方崩溃
反模式： 重启从 step 0 开始
正确： 跳过已完成 step
⚙️
资源控制
内存泄漏每小时 +5MB，
24 小时 = 120MB
反模式： 无限积累状态
正确： 每 checkpoint 周期清理
Design Note
cron 解决「什么时候跑」· checkpoint 解决「跑到哪了」· content-hash 解决「这个有没有跑过」 三者组合，才是生产级长任务的完整解

CREATE TABLE task_state (
  task_id      TEXT PRIMARY KEY,
  status       TEXT NOT NULL, -- 状态枚举
  checkpoint  TEXT,           -- JSON
  progress     INTEGER,         -- 0-100
  total_steps &span class="code-string">INTEGER,
  error_msg    TEXT,
  updated_at   TEXT NOT NULL
);

-- task_results：每步骤结果（幂等）
CREATE TABLE task_results (
  task_id      TEXT,
  step_id      TEXT,
  result_data  TEXT,          -- JSON
  completed_at TEXT,
  UNIQUE(task_id, step_id) -- 幂等保证
);
status 枚举
pending 已创建，未开始
running 执行中
checkpoint_saved 步骤已落盘
completed 全部完成
failed 执行失败
# 崩溃恢复关键函数
def load_completed_steps(task_id):
  rows = db.execute(
    "SELECT step_id FROM task_results WHERE task_id=?",
    (task_id,)
  ).fetchall()
  return {row[0] for row in rows}

import hashlib

def get_cache_key(voice, text):
  """相同声音 + 相同文本 = 相同 key"""
  content = f"{voice}:{text}"
  return hashlib.sha256(
    content.encode()
  ).hexdigest()

def synthesize(voice, text):
  key = get_cache_key(voice, text)
  path = CACHE / f"{key}.wav"

  if path.exists():
    return path.read_bytes() # cache hit!

  audio = tts.synthesize(voice, text)
  path.write_bytes(audio)
  return audio
核心洞察
content-hash 缓存同时解决两个问题：
1. 跨天幂等 — 同一段文字明天和今天一样，不浪费 GPU
2. 断点续传 — 重启后查缓存，已合成的直接复用
