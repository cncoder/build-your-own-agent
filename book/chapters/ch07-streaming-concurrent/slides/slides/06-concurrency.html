CLAUDE_CODE_MAX_TOOL_USE_CONCURRENCY = 10

可通过环境变量覆盖。但调高前先考虑以下三个设计理由。

为什么是 10，不是 100？
理由 1

资源保护

同时开 100 个 bash 进程或文件操作，会耗尽系统文件描述符（fd limit）。10 是大多数系统的安全上界。

理由 2

防 API Rate Limit

如果工具调用外部 API（web_search、数据库、第三方服务），并发太高触发 429 Too Many Requests 反而更慢。

理由 3

控制工具副作用

写文件、执行 shell 命令等有副作用的工具并发执行可能产生竞争条件（race condition）。10 的上限给人工介入留出空间。

Python 实现：asyncio.Semaphore(MAX_CONCURRENT_TOOLS)
语言不同，思想一致。
