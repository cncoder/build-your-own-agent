DeepSeek reasoning_content — 两路流式分离 
# DeepSeek-R1 的思维链和正文是两个独立字段
async for chunk in stream:
    delta = chunk.choices[0].delta

    # 思维链（用于调试记录）
    reasoning = delta.get("reasoning_content") or ""
    if reasoning:
        current_reasoning += reasoning

    # 正文（展示给用户）

两路分开处理：reasoning_content 记录供调试，content 流式输出给用户。混淆会把推理过程直接暴露给用户。
三家通用 cache token 字段统一（llm.py:18-38） 
def _parse_usage(usage, provider):
    if provider == "anthropic":
        cached = usage.get("cache_read_input_tokens", 0)
    elif provider == "deepseek":
        cached = usage.get("prompt_cache_hit_tokens", 0)
    else:  # openai
        details = usage.get("prompt_tokens_details", {})
        cached = details.get("cached_tokens", 0)
    return cached
cache_read_input_tokens ← Anthropic
prompt_cache_hit_tokens ← DeepSeek
prompt_tokens_details.cached_tokens ← OpenAI
这 20 行代码让你的 agent 支持三家 provider 的 prompt cache 统计，监控成本时不会漏算。
