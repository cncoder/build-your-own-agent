SSE 协议解析

Server-Sent Events — LLM 流式输出的通用选择

# HTTP 响应头
Content-Type: text/event-stream
Cache-Control: no-cache

# 每条消息：event + data + 空行
event: content_block_delta
data: {"type":"content_block_delta","delta":{"type":"text_delta","text":"你好"}}

event: content_block_delta
data: {"type":"content_block_delta","delta":{"type":"text_delta","text":"世界"}}

# 流结束
event: message_stop
data: {"type":"message_stop"}

# OpenAI 约定的结束标志
data: [DONE]
规则 1
空行分隔消息
每条 SSE 消息由一行或多行 key: value 组成，以空行结束。客户端逐行读取，遇空行触发事件处理。
规则 2
data: 字段是 JSON
LLM API 的 data 字段是 JSON 字符串，需要 json.loads() 解析。[DONE] 是纯文本标志（OpenAI 约定），不是 JSON。

SSE（选择原因）

  • 纯 HTTP，代理友好
  • 单向推送，足够用
  • 断线自动重连
  • 无需特殊库

WebSocket（为何不用）

  • 双向全双工，过于复杂
  • 需要握手建立连接
  • 代理服务器有兼容问题
  • LLM 生成是单向的
