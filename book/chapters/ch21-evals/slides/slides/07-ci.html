CI 集成：每个 PR 自动跑 Eval

PR 跑 20 条（快），main 跑 100 条（精）。composite < 0.75 阻断合并。

# .github/workflows/eval.yml
name: Lena Eval CI

on:
  pull_request:
    branches: [main]
  push:
    branches: [main]

env:
  EVAL_SAMPLE_SIZE: 20      # PR 只跑 20 条
  EVAL_FULL_SIZE:   100     # main 跑全量
  EVAL_PASS_THRESHOLD: 0.75 # 低于此值阻断

jobs:
  eval:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with: { python-version: "3.12" }
      - run: pip install -r code/lena-v1.7/requirements.txt

      - name: Run eval
        env: { ANTHROPIC_API_KEY: ${{ secrets.ANTHROPIC_API_KEY }} }
        run: |
          python code/lena-v1.7/run_eval.py \
            --dataset golden-dataset.json  \
            --sample-size $EVAL_SAMPLE_SIZE \
            --threshold   $EVAL_PASS_THRESHOLD

      - name: Comment on PR
        uses: actions/github-script@v7
        # 自动在 PR 页面写 eval 结果评论
🔀PR 提交
开发者提交代码，CI 自动触发
⚡Eval 运行（20 条采样）
LLM judge + 三维度评分，约 30 秒
💬PR 评论
自动写入质量报告：composite 分、各维度、通过/失败
✅PASS → 可合并
composite ≥ 0.75，无退化检测
❌FAIL → 阻断合并
composite < 0.75 或有回归，必须修复再提交
