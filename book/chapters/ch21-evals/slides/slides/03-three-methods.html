三种 Eval 方法

各有优缺点，组合使用，覆盖不同场景

🤖
LLM-as-judge
让另一个模型来打分
优点能评主观质量，处理开放式输出
优点比人工快 100x，可扩展
缺点judge 也会犯错，有偏向
缺点每次调用有 API 成本
适用场景
开放式 QA、内容生成、多步推理
🏆
Golden Dataset
黄金标准用例库
优点完全可重复，零 API 成本
优点适合精确回归检测
缺点构建成本高（100条需数天）
缺点格式变化时测试易失效
适用场景
工具调用参数、Safety 规则、固定格式输出
📈
Regression Suite
与历史版本对比
优点不需要期望值，检测相对退化
优点模型升级/重构的必备工具
缺点需要维护 baseline 版本
注意退化阈值设定需要经验
适用场景
模型升级前后对比、重构回归检测
