两大运维血坑 — 别重复踩

来自生产运维经验 · CLAUDE.md lessons-learned.md

坑 01 · systemd
StartLimitBurst 把 Lena 永久停了
AWS key 过期 → Lena 启动即崩溃 → 5 次后 systemd 触发 StartLimitBurst → 进程被永久暂停。 整个下午都没发现 Lena 挂了——没有任何 Discord 告警，因为 Watchdog 也是 Lena 本身的任务，Lena 死了 Watchdog 也死了。
# 手动恢复命令（记住它！）
sudo systemctl reset-failed lena
sudo systemctl start lena
解法 01 · systemd
独立 Watchdog + 独立告警通道
两个结构性解法： ① 放宽 StartLimitBurst（300s/5次 → 容忍偶发错误） ② 独立 Watchdog 进程，用独立的 Telegram bot 告警。 Watchdog 不依赖 Lena，Lena 全线瘫痪时 Watchdog 仍然存活并发出告警。
# lena.service
StartLimitIntervalSec=300
StartLimitBurst=5
坑 02 · launchd
SIGUSR1 不刷新 plist 环境变量
修改了 plist 里的 环境变量，发 SIGUSR1 希望 让 launchd 重读配置——环境变量完全没变。 原因：SIGUSR1 只通知进程「配置可能变了」， 但 launchd 本身不重读 plist 文件。
# 正确做法：每次改 plist 后
launchctl unload ai.lena.agent.plist
launchctl load ai.lena.agent.plist
解法 02 · launchd
ThrottleInterval 防重启风暴
launchd KeepAlive 不加 ThrottleInterval 时， 进程崩溃会以最快速度重启。 60 秒内触发 5 次后 launchd 把进程永久暂停（同 StartLimitBurst 症状）。 加了 60s 节流后，崩溃 → 等 60s → 重启，有时间记录日志、发告警。
<key>ThrottleInterval</key>
<integer>60</integer> <!-- 必须！-->
