Lena 的部署选型

三种方案没有最好，只有最合适

Mac 开发者首选
launchd
macOS · ~/Library/LaunchAgents
随用户登录自动启动，系统级守护，无需额外依赖。 个人 7×24 AI 助手就是这样运行的——关了终端 Lena 还在，重启 Mac Lena 自动回来。
适用：Mac 个人 7×24 AI 助手
systemd
Linux · /etc/systemd/system
Linux 服务器标准方案，journald 日志集成， restart 策略灵活。VPS / EC2 部署 Lena 的标准选择， 记得配好 StartLimitBurst。
适用：Linux 服务器 · EC2 · VPS
Docker Compose
多服务 · Lena + Redis + Jaeger
多个服务组合时的最佳方案，服务间网络隔离， healthcheck 依赖，资源限制精细控制。 生产级多服务部署的标准起点。
适用：多服务架构 · 需要 Redis/监控
云函数（Lambda/Cloud Functions）什么时候更好？ 当 Lena 是「按需触发」而非「持续运行」时——比如每天一次日报任务， 用 EventBridge Cron → Lambda 比一直运行的服务更省钱。 两种范式不互斥：Lena 核心用 systemd/launchd，一次性任务用 Lambda。
