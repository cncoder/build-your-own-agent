OpenTelemetry · 四概念

Trace / Span / Metric / Log — 一图理清

三者通过 trace_id / span_id 关联，从单条日志可追溯完整调用链

Trace
一次完整用户请求的全旅程。 有唯一 trace_id， 跨越所有服务边界，从入口到出口。
trace_id = "8d4e7a2b..."
从 "用户发出消息"
→ 到 "Lena 返回最终回复"
Span
Trace 内的单个操作，有开始/结束时间、 属性键值对，可以有父子关系，形成树状结构。
span_id = "f3c1..."
parent = "8d4e..."
duration_ms = 2100
model = "claude-sonnet-4-6"
Metric
数值随时间的变化，三种类型： Counter（累加）， Gauge（当前值）， Histogram（分布）。
Counter: 总 LLM 调用次数
Gauge: 当前 token 使用量
Histogram: P99 延迟分布
Log
单个事件的快照。结构化日志（structlog JSON） 带 trace_id 和 span_id， 可在 Trace 详情里展开。
{
"event": "llm_call",
"trace_id": "8d4e...",
"span_id": "f3c1..."
}
structlog JSON 日志 → 带 trace_id / span_id 关联 → 从日志跳到完整调用链 → Jaeger UI · AWS X-Ray · Honeycomb
