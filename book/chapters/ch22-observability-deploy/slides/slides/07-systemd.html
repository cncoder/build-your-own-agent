lena.service 完整配置 — 血泪教训：StartLimitBurst

崩溃重启过快被 systemd 限制 → 进程永久停止 → 用 systemctl reset-failed 恢复

# /etc/systemd/system/lena.service [Unit] Description=Lena AI Agent v1.8 After=network.target [Service] Type=simple User=lena WorkingDirectory=/opt/lena ExecStart=/opt/lena/.venv/bin/python -u -m src.main Environment=AWS_REGION=us-west-2 EnvironmentFile=/opt/lena/.env Restart=on-failure RestartSec=10s # 血泪教训：防重启风暴（默认 5次/10s 触发后进程永久停止） StartLimitIntervalSec=300 StartLimitBurst=5 MemoryMax=2G CPUQuota=80% # 安全加固 NoNewPrivileges=yes PrivateTmp=yes StandardOutput=journal # journalctl -u lena -f [Install] WantedBy=multi-user.target
$ sudo systemctl daemon-reload 重载 unit 文件
$ sudo systemctl enable --now lena 启用 + 立即启动
$ sudo journalctl -u lena -f 实时日志
$ sudo journalctl -u lena -o json | head JSON 格式（jq 可查）
$ sudo systemctl reset-failed lena StartLimitBurst 触发后恢复！
🩸 真实案例（CLAUDE.md lessons-learned.md §AWS systemd）
AWS key 过期 → Lena 启动即崩溃 → 5 次后 systemd 把她彻底停了 → 没有任何 Discord 告警 → 整个下午都没发现 agent 挂了。
解法 ①：增大 StartLimitIntervalSec/Burst（容忍偶发错误）
解法 ②：独立 Watchdog（不归 systemd 管）检测存活 + Telegram 独立告警
