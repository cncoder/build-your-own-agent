结构化日志 · structlog

为什么 JSON 日志让排查快 10 倍

文本日志 vs. 结构化日志的本质差异

文本日志（Bad）
# 三天后你根本不知道这是哪次调用 print(f"调用 LLM，结果：{result}") # 想查今天花了多少钱？ # grep -A5 "2026-05-05 03:12" # 然后肉眼一行一行读……
查某次调用 token 肉眼 grep 读
统计今日花费 无法直接统计
找所有失败工具调用 grep ERROR 乱七八糟
按模型聚合延迟 写脚本解析字符串
结构化日志（Good）
import structlog log = structlog.get_logger() log.info("llm_call", model="claude-sonnet-4-6", tokens=1234, latency_ms=2100, span_id="a1b2c3")
查某次调用 token jq '.span_id=="a1b2c3"'
统计今日花费 jq '[.cost_usd] | add'
找所有失败工具调用 jq '.event=="tool_fail"'
按模型聚合延迟 jq -r '.model + "," + (.latency_ms|tostring)'
Design Note：开发环境用 ConsoleRenderer（彩色可读）， 生产环境用 JSONRenderer，一个配置项切换，零改代码。
