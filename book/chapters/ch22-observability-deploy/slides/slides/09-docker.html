Docker Compose · 多服务部署

Lena + Redis + Jaeger — 一键启停整套服务

服务间网络隔离（lena-net）· healthcheck 依赖 · restart: unless-stopped

# docker-compose.yml services: lena: build: . restart: unless-stopped env_file: .env environment: - OTEL_EXPORTER_OTLP_ENDPOINT=http://jaeger:4317 - REDIS_URL=redis://redis:6379 depends_on: redis: condition: service_healthy deploy.resources.limits: memory: 2G cpus: "1.5" redis: image: redis:7-alpine healthcheck: test: ["CMD", "redis-cli", "ping"] jaeger: image: jaegertracing/all-in-one:1.55 ports: - "16686:16686" # Jaeger UI - "4317:4317" # OTLP gRPC
lena-net · bridge 隔离网络
lena Agent 主进程
structlog + OTel + Budget 熔断 :8080 /health
redis 会话状态 / Budget 跨实例共享
appendonly yes 持久化 :6379
jaeger OTel 追踪收集 + 可视化 UI
生产替换为 AWS X-Ray :16686 UI · :4317 OTLP
$ docker compose up -d 启动所有服务
$ docker compose logs -f lena 实时日志
$ docker compose restart lena 重启单个服务
