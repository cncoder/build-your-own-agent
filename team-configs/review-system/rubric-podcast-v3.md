# Podcast Rubric v3

针对 `book/chapters/chXX-*/podcast.md` 的评分规则。

实现：`scripts/score_podcast_v3.py`

## A 组 · 结构硬铁律（本地 regex）

每项权重 0.05，必须 = 10。

### D1 主讲占比

- 涛哥字数 ≥ 95% → 10
- ≥ 92% → 7
- ≥ 90% → 4
- < 90% → 0

一票否决 V1/V10：< 90% 或 周迅 ≥ 5%。

### D2 副讲人单段字数

周迅任何一段 > 50 字 → 扣分；> 120 字 → 触发 V2 否决。

### D3 段长

任何段落 > 290 字 → 扣分；> 320 字 → 触发 V9（TTS chunk 硬限）。

### D4 禁词禁格式

正则匹配下列全部不出现：
- `Let's` + 中文（V7）
- `So here we go` / `That's it` / `Now it is time` / `Along the way`
- `本章 arc` / `本节讨论` / `接下来我们将`
- `Abel` 人名（V3）
- `/Users/` `~/code/` `~/.claude/`（V4）
- `R1-R99` / `R7-G` 格式代号（V8）

## B 组 · 内容质量（LLM 判官）

### D5 术语讲清率（权重 0.15，门槛 ≥ 7.5）

从 README 抽 10 个核心术语，每个三档：
- 讲清（1.0）= 有定义 + 作用 + 例子 三要素中 ≥ 2
- 仅提及（0.3）= 名字在文本但无解释
- 没讲（0）= 字符串都不出现

Score = 10 个术语平均 × 10。

### D6 事实准确（权重 0.25，门槛 ≥ 8.5）

**最高权重 + 最高门槛**。对照 README 逐段核事实：
- 类名 / 方法名 / 参数 / 默认值 / 数字 / 引用是否一致
- 是否编造 README 没讲的内容
- 是否歪曲 README 的说法

每个错误扣 0.5 - 2.0 分。编造内容最重。

### D7 逻辑连贯（权重 0.15，门槛 ≥ 7.5）

- 段间过渡是否自然
- 是否有突兀引入的概念
- 论证链是否清晰（动机 → 问题 → 方案 → 细节 → 收尾）
- 副讲人提问是否机械化（"防线X？" 这种目录朗读感扣分）

### D8 信息密度（权重 0.15，门槛 ≥ 7.5）

每段打标签：新信息 / 复述 / 灌水 / 堆砌。

Score = 新信息段 / 总段 × 10 − 堆砌超额惩罚（D ≥ 3 额外 −1.5）。

### D9 可复述性（权重 0.10，门槛 ≥ 7.5）

扮后端工程师读者听完独立复述 5 条核心命题：
- 完整复述 2.0
- 部分复述 1.0
- 只记得名字 0.3
- 没印象 0

Score = 5 条之和 × 1.0（满分 10）。

## 权重和门槛

```
A 组合计权重：0.20
B 组合计权重：0.80

Pass = 加权总分 ≥ 9.1
     AND A 组每项 = 10
     AND B 组每项 ≥ 7.5
     AND D6 ≥ 8.5
     AND 0 个 V1-V11
```
